📖 ファイル構成の説明

発展的な構成について詳しく説明します。

なぜこの構成を使うの？

1. 共通のスタイルを再利用できる

common.css に書いたスタイルは、すべてのページで使えます。

index.html：

<link href="assets/css/common.css">

about.html：

<link href="assets/css/common.css">

両方のページで同じナビゲーションバーが使えます！

2. ページごとに違うスタイルを適用できる

ホームページには home.css、説明ページには about.css を使います。

3. ファイルを探しやすい

すべての資材が assets/ フォルダにまとまっているので、どこに何があるか分かりやすい！

📝 パスの参照方法

HTMLからCSSを読み込む

<link rel="stylesheet" href="assets/css/common.css">

index.html → assets → css → common.css

HTMLからJavaScriptを読み込む

<script src="assets/js/common.js"></script>

index.html → assets → js → common.js

HTMLから画像を読み込む

<img src="assets/images/photo.jpg">

index.html → assets → images → photo.jpg

📊 構成の比較

構成 特徴 向いているプロジェクト
基本的 すべて同じフォルダ 小規模（1-3ページ）
整理された css/, js/, images/で分類 中規模（3-10ページ）
発展的 assets/で統一、ファイルを細分化 大規模（10ページ以上）
