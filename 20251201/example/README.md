# サンプルプロジェクト

このフォルダには、3つの異なるファイル構成のサンプルが入っています。
それぞれのサンプルを開いて、ファイル構成の違いを比べてみよう！

---

## 📁 サンプルの種類

### 1. basic/ - 基本的な構成

**フォルダ構成：**
```
basic/
├── index.html
├── style.css
└── script.js
```

**特徴：**
- すべてのファイルが同じフォルダにある
- 小規模なプロジェクトに最適
- ファイルの読み込みが簡単

**こんなときに使おう：**
- 1ページだけのウェブサイト
- 練習用のプロジェクト
- 簡単なアプリ

**開き方：**
1. `example` → `basic` フォルダを開く
2. `index.html` をダブルクリック（自動的にブラウザが開きます）
3. ボタンを押してみよう！

---

### 2. organized/ - 整理された構成

**フォルダ構成：**
```
organized/
├── index.html
├── css/
│   └── style.css
├── js/
│   └── script.js
└── images/
    └── （画像ファイル）
```

**特徴：**
- ファイルの種類ごとにフォルダで整理
- 中規模なプロジェクトに最適
- どこに何があるか分かりやすい

**こんなときに使おう：**
- 3-10ページくらいのウェブサイト
- 画像やファイルが増えてきたとき
- チームで作業するとき

**開き方：**
1. `example` → `organized` フォルダを開く
2. フォルダの中に `css` や `js` フォルダがあることを確認
3. `index.html` をダブルクリック
4. カウンターアプリが表示されます！

**ポイント：**
- パスは `css/style.css` のようにフォルダ名も含める

---

### 3. advanced/ - 発展的な構成

**フォルダ構成：**
```
advanced/
├── index.html
├── about.html
└── assets/
    ├── css/
    │   ├── common.css
    │   ├── home.css
    │   └── about.css
    ├── js/
    │   ├── common.js
    │   └── home.js
    └── images/
        └── （画像ファイル）
```

**特徴：**
- 複数のページを持つ
- 共通のスタイルとページごとのスタイルを分けている
- すべての資材を `assets` フォルダにまとめている
- 大規模なプロジェクトに最適

**こんなときに使おう：**
- 10ページ以上のウェブサイト
- 複雑な機能を持つアプリ
- 本格的なプロジェクト

**開き方：**
1. `example` → `advanced` フォルダを開く
2. 複数のHTMLファイルと `assets` フォルダがあることを確認
3. `index.html` をダブルクリック
4. ページが開いたら、上のナビゲーションバーから「説明」ページにも行ってみよう
5. カードをクリックしてみよう！

**ポイント：**
- パスは `assets/css/common.css` のように深くなる
- 共通のスタイルを複数のページで再利用できる

## 🎯 学習の進め方

### 🌟 初めての人向け

**まず全部開いてみよう！**

1. `basic` フォルダの `index.html` をダブルクリック → ブラウザで開く
2. `organized` フォルダの `index.html` をダブルクリック → ブラウザで開く
3. `advanced` フォルダの `index.html` をダブルクリック → ブラウザで開く

3つのタブができて、それぞれ違うページが開きましたか？
これが3つの異なる「ファイル構成」です！

---

### ステップ1：基本的な構成を見る

1. `basic` フォルダを開く
2. 3つのファイル（index.html, style.css, script.js）があることを確認
3. `index.html` を右クリック → メモ帳で開く
4. どうやって他のファイルを読み込んでいるか見てみよう

### ステップ2：整理された構成を見る

1. `organized` フォルダを開く
2. `css` フォルダと `js` フォルダがあることを確認
3. `index.html` を右クリック → メモ帳で開く
4. パスの書き方（`css/style.css`）に注目しよう

### ステップ3：発展的な構成を見る

1. `advanced` フォルダを開く
2. 複数のHTMLファイルと `assets` フォルダがあることを確認
3. `index.html` と `about.html` の両方を開いて比べてみよう
4. どちらも同じCSSファイルを使っていることを確認しよう

### ステップ4：実際に編集してみる

**編集の手順：**
1. 好きなサンプルを選ぶ
2. ファイルを右クリック → メモ帳で開く
3. テキストや色を変えてみる
4. 保存する（Ctrl+S または Cmd+S）
5. ブラウザで更新する（F5キー）

**試してみよう：**
- タイトルを変える
- 色を変える
- 新しいテキストを追加する

## 🔍 よくある質問

### Q1: どの構成から始めればいい？

**A:** 最初は **basic** から始めましょう！
慣れてきたら、**organized** や **advanced** に挑戦してみてください。

### Q2: フォルダの名前は変えてもいい？

**A:** はい、変えてもOKです！
ただし、HTMLの `href` や `src` も同じように変える必要があります。

例：
- `css/` を `styles/` に変えたら
- `<link href="styles/style.css">` に変える

### Q3: 自分のプロジェクトはどの構成にすればいい？

**A:** プロジェクトの大きさで決めましょう：
- **小さい**（1-3ページ）→ **basic** または **organized**
- **中くらい**（3-10ページ）→ **organized**
- **大きい**（10ページ以上）→ **advanced**
